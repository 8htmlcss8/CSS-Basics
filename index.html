CSS Basics

You're the web artist. So... why should a CSS framework restrict your artistry?

CSS Basics is built to start styling your page without worry about any position and behaviour problems. It's mobile first and scales up to large displays with Twitters incredible Bootstrap grid system and responsive utilities.

Setup

Just include basics.min.css before all other stylesheets and javascripts.

Features

  • No styles - just basics
  • Responsive and adaptive
  • Mobile first
  • RTL support
  • Easy implementation - just a few classes
  • Lightweight - only ~21 KB (minimized version)

Included

  • Menus
    • Horizontal
    • Vertical
  • Forms
  • Tables
  • Helpers
  • Positioning
  • Externals
    • Normalize.css
    • Twitters Bootstrap grid system and responsive utilities
    • WordPress core (modified) - Of course you can use it outside of WordPress

Browser support

  • IE >= 9
    IE < 9 doesn't support these requirements:
    Media Queries
    Affects: All responsive utilities
    Solution: Include respond.js
    2D Transform
    Affects Positioning
    Solution: Include basics.ie.css - Requires to set width/height
  • Chrome, Firefox, Safari and all other major browsers

Forms

Love the way to wrap form elements into <label>'s. Advantage: no need to assign ID's to every form element. If you want to style only the text, wrap it with <span>.

See the responsive behaviour by resizing your browser window - it's never been easier to make forms mobile ready!

Default behaviour - everything is horizontal - mobile first
Inline form

Set .form-inline- (with breakpoint keyword) to a parent element (form, fieldset or label etc.) to make everything in-line. Smaller breakpoints will overwrite larger breakpoints. Class .form-inline-xs will force in-line on mobile devices.

Table like

Form elements should not be structured with tables. To make a form looks like a table, use bootstraps grid system. You can use <label>'s as grid row, still don't worry about ID's - cool.

Helpers

Hidden

Hide elements with .hidden (inaccessible for screen-readers)

.hidden {
  display: none !important;
  visibility: hidden !important;
}

Invisible

Hide elements with .invisible (accessible for screen-readers)

Also works with .screen-reader-text for compatibility reasons with WordPress.

.invisible {
  position: absolute !important;
  top: -9999px !important;
  left: -9999px !important;
}

Clearfix

Clear floats with .clearfix

Alignments

Align text and images with .alignleft, .alignright, .aligncenter and .alignnone.

Image caption

Gice your images an caption below the image with .caption.

Still works with .wp-caption for compatibility reasons with WordPress.

Positioning

This solution works whether you set width/height or not to the positioned element. Take note that outer elements will overlap the border while inner elements won't. Class structure is [x-property]-[y-property] e.g. right-bottom.

Important!

Since IE<=8 doesn't support CSS3 transform, you have to include basics.ie.css. Then you have to set width/height to the elements.

Position reset

In order to reset all position effect, set .position-reset- (with responsive keywords). This will reset it only on the specified keyword-width. Reset style code: 

	
position: static;
top: auto;
left: auto;
right: auto;
bottom: auto;
-webkit-transform: none;
   -moz-transform: none;
    -ms-transform: none;
     -o-transform: none;
        transform: none;
